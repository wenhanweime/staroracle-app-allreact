# 简介
“集星问问 (StarO)”是一款 AI 引导式自我觉察产品，其核心概念是将用户每一次深度的问答和反思，具象化为一个可交互、会演化的**“心灵宇宙生态系统”**。在这个系统中，用户的觉察卡片以“星星”+“星卡”的形态存在。
# 关键词
银河、星卡、点亮、对话、星星升级、自我觉察
# 架构
本项目采用 “瘦客户端，富后端” 的云端主导架构：
• 前端： Swift 客户端，负责高性能渲染（Metal/3D 星空）和对话前端框架、银河系渲染、星卡渲染、用户交互、优雅流畅的动画
• 后端： Supabase (PostgreSQL + RLS) 作为数据主真相源 (Source of Truth)，Edge Functions (Deno/TypeScript) 承担复杂的对话管理、LLM 协调和觉察数据、星星数据的联合等所有业务逻辑。
# 为什么做 (Why are we doing this?)
本项目旨在解决自我觉察和情绪记录领域的几个核心障碍，特别是缺乏正向激励和成长不可见的问题：
1. 消除高门槛： 通过点击直接弹出卡片的 “摘星模式” 提供低成本的对话话题，帮助用户了解心理学、觉察知识（包含【问题】【答案之书】【书籍摘录语录】等的卡片），解决了用户“空有情绪，却不知从何说起”的**“空白页综合症”**。
2. 提供微反馈： AI 扮演**“思想的助产士”的角色，通过 、精细化震动反馈和“文字转星”动画，提供即时的感官和情感奖励，让“觉察 = 收获”**变得直观、有感。
3. 构建深度记忆： 将 NextChat 的**“智能记忆管理”逻辑移植到后端服务，实现系统提示词、长期记忆摘要和短期消息截断的分层记忆架构**，确保深度对话的连续性。
4. 直接对话：可以跟星谕聊任何事情，直接开始聊也可以点亮星星，这是比摘星门槛略高（需要输入文字）但更自然的方式。
5. 银河系：摘星点击可以点亮，对话可以点亮，最终成为灿烂银河。
6. 星卡：摘星、对话都可以产生星卡，并且随着是否有有深度的觉察对话产生，会有不同等级的星卡产生出来。详细等级查看技术文档。
# 做完会怎样 (What will happen when it's done?)
项目完成后，将构建一个具备内在生命力和演化逻辑的“心灵宇宙”，实现以下关键成果：
1. 千人千面的宇宙： 所有用户的星图都基于**“三大悬臂”（情绪、关系、成长星域）的统一骨架，但星星的位置、密度和形状将由用户的个人觉察内容和顺序决定，形成独一无二的“心理地图”**。
2. 可视化成长闭环： 产品将实现从 “被动接收灵感”（摘星） 到 “主动自我探索”（铸星） 的完美闭环。星星的演化状态（星辰、新星、恒星、超新星、白矮星）由单次生成时的深度和用户的主动回顾行为决定，而不是通过“持续打卡”来维持，从而降低用户压力。
3. 云端可靠性： 基于 Supabase 的数据隔离和行级安全（RLS），确保用户数据私密和安全，天然支持 Swift 客户端在多设备间的聊天记录和星图状态的实时同步。
项目最终将从一个简单的应用升级为一个具备灵魂、能够陪伴用户一生的**“数字生命体”**。
